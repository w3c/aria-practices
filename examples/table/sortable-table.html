Sortable Table Example

The example below illustrates an implementation of the WAI-ARIA table design pattern for a table with sortable rows. The example uses HTML table markup to represent the rows, caption, and header and data cells of the table. The aria-sort attribute is added to the column header of the data used to sort the table. A button element in the each sortable column header is used to change the sorting column and order. The data values in the columns are either all text or all numbers and a class name is used to indicate (e.g "num") which columns should be sorted as a number value. One column (e.g. "address") in the example table is not used to sort the table.

Accessibility Features

  • HTML table markup is used to create the rows, caption, and to identify the column headers for the data cells. The use of HTML supports the ARIA principle of using native element semantics whenever possible.
    NOTE: The use of the thead and tbody elements to separate the header rows from the sortable data rows helps reduce the complexity of the sorting script.
  • Since the each column header cell and it's associated sorting button share the same accessible name, aria-describedby reference on the button element is used to provide additional information on the action of the button. This supports table navigation commands of screen readers to just provide the column header name when moving between columns and not include additional information about the button action.
  • Uses CSS :hover and :focus pseudo-class for styling visual keyboard focus and hover for the sorting button.
    • The focus indicator encompasses both the column label and sort direction icon to make it easier to perceive when the switch has focus.
    • The hover indicator is the same as focus indicator to make it easier to perceive that clicking either the column label or the sort direction icon will activate the sorting the table. Including both the column label and the sort direction icon maximizes the target size of the sorting button to make it easier to activate with pointing devices for people with visual and movement impairments. CSS positioning and sizing is used to make the button fill most of the header cell area to maximize the target size to activate the button with a pointing device.
    • The cursor is changed to a pointer when hovering over the button to help people with visual impairments identify it as an interactive element.
  • To ensure the sorting direction icon is synchronized with the value of the aria-sort attribute, CSS selectors based on the aria-sort value are used to render the sort direction icon.
  • When a sortable column is un-sorted a diamond shaped icon (e.g. "â™¢" ) is used in the column to provide a visual affordance that the column is sortable. The use of a different shaped icon supports the user in identifying the difference with the sorted column icon in high contrast modes.
  • To ensure the sorting direction icons have sufficient contrast with the background when high contrast settings invert colors a character entity is used to indicate the sorting direction.

Role, Property, State, and Tabindex Attributes

Role Attribute Element Usage
aria-sort="value" th
  • The aria-sort attribute is only defined on the column that is sorted. As the sorted columns are changed the aria-sort attribute is removed from un-sorted columns.
  • A value of "ascending" identifies the data cells in the column are sorted in ascending order.
  • A value of "descending" identifies the data cells in the column are sorted in descending order.
aria-describedby="IDREF" button
  • Provides an accessible description for each button used for sorting (e.g. "Sort in descending order").
  • The descriptions for all the button elements are updated each time any button is activated to indicate any changes in action of each button.
aria-hidden="true" span Removes the icon character from being part of the accessible name of the button. The direction of the sort is defined using aria-sort attribute on the th element.

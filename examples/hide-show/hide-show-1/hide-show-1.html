Role, Property, State, and Tabindex Attributes

Role Attribute Element Usage
button div
  • The role="button" attribute identifies the div element as a ARIA button.
  • The accessible name comes the ???.
  • The button widget needs a tabindex="0" value.
tabindex="0" div
  • The div["button"] is identified as an interactive element and is added to the tab order of the page by setting the tabindex="0".
aria-controls="[IDREFS]" div
  • A list of IDREFs identifies which checkboxes are controlled by the mixed checkbox.
aria-expanded="false" div
  • Indentifies the button button as hiding content identified with aria-controls property.
  • This state reflects that all the controlled checkboxes are unchecked.
  • CSS attribute selectors (e.g. [aria-expanded="false"]) are used to synchronize the visual states with the value of the aria-expanded attribute.
  • The CSS :before psuedo selector and the content property are used to indcate visual state of "expanded" to support high contrast setting in operating systems and browsers.
aria-expanded="true" div
  • Indentifies the button button as showing content identified with aria-controls property.
  • This state reflects that all the controlled checkboxes are checked.
  • CSS attribute selectors (e.g. [aria-checked="true"]) are used to synchronize the visual states with the value of the aria-checked attribute.
  • The CSS :before psuedo selector and the content property are used to indcate visual state of "checked" to support high contrast setting in operating systems and browsers.

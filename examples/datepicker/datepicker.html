Datepicker Examples

The following command and input examples demonstrate the datepicker design pattern.

Example

Keyboard Support

Date Input keyboard

Key Function
Down Arrow Open the calendar dialog

Calendar Button

Key Function
Space,
Return
  • Toggles calendar dialog.
  • NOTE: This button has been removed from tab order of the page by setting tabindex=-1

Next/Previous Buttons (for month and year)

Key Function
Space,
Return		 Change the month for selecting a date.
ESC Close the dialog and move focus back to the calendar button.
TAB Move focus to next button or grid inside the dialog.
Shift +
TAB		 Move focus to previous button or grid inside the dialog.

Calendar Grid

Key Function
Space,
Return		 Select the date, close the dialog box and move focus to the calendar button.
Up Arrow Move the focus to the same day of the previous week.
Down Arrow Move the focus to the same day of the next week.
Right Arrow Move the focus to the next day.
Left Arrow Move the focus to the previous day.
PageUp
  • Change the calendar to the previous month.
  • If current day of month available on the new month, focus will be on that date, otherwise focus will be on the first date of the month.
PageDown
  • Change the calendar to the next month.
  • If current day of month available on the new month, focus will be on that date, otherwise focus will be on the first date of the month.
TAB Move focus to next button inside the dialog.
Shift +
TAB		 Move focus to previous button inside the dialog.
ESC Close the dialog and move focus back to calendar button

OK and Cancel Buttons

Key Function
Space,
Return
  • If it's "Cancel" button, closes the calendar dialog, moves focus to calendar button, does not update date in date input.
  • If it's "OK" button, closes the calendar dialog, moves focus to calendar button, does update date in date input.
TAB Move focus to next button inside the dialog.
Shift +
TAB		 Move focus to previous button or grid inside the dialog.
ESC Closes the calendar dialog, moves focus to calendar button, does not update date in date input.

Role, Property, State, and Tabindex Attributes

Role Attribute Element Usage
status span Identifies the element that serves as a form of live region.
aira-control="IDREF" button Whenever the button clicked or pressed, the status of the calendar will also change.
aira-label="IDREF" button Define the function of the current element.
grid table
  • Identifies the element that serves as the grid widget container.
  • Since the grid role is applied to a table element, the row , rowheader, colheader, and gridcell roles do not need to be specified because they are implied by tr, th, and td tags.
aira-label="IDREF" th
  • Define a string to the current element.
dialog div Identifies the element that serves as the dialog container
aria-labelledby="IDREF" div Gives the dialog an accessible name by referring to the element that provides the dialog title.
button
  • Identifies the element as a button widget.
  • Accessible name for the button is defined by the text content of the element.
tabindex="0" button
  • Includes the element in the tab sequence.
  • The current button is focused
tabindex="-1" button Exclude the button from tab sequence but it is focusable.
aria-selected="false" button
  • Identifies the button as a selectable button.
  • Indicates the button is not selected.
aria-selected="true" button
  • Identifies the button as a selectable button.
  • Indicates the toggle button is selected .
aria-expanded="true" button Indicates the parent node is open, i.e., the descendant elements are visible.

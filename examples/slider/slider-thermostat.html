Slider Examples with aria-orientation and aria-valuetext

The following example of a hypothetical online thermostat demonstrates the slider design pattern. The desired temperature is set with the first slider, which is vertically oriented. Two horizontal sliders set fan speed and mode. The horizontal sliders illustrate how to use aria-valuetext to provide assistive technologies with meaningful names for numeric values.

Accessibility Features

  • A border is placed around the slider and related controls when the slider or a related element receives focus.
  • The placement of the slider value to the right of the thumb of the vertical supports users with low vision by allow them to more easily view the value of the slider as they move the thumb.
  • A circle SVG image is used for indicating the slider with keyboard focus.
  • The use SVG images enables the thumb, focus ring and rail of the slider to adapt to operating system high contrast color schemes and settings.
  • The CSS currentColor value for the fill and stroke properties for the rail, thumb and slider supports user selected color schemes in operating system high contrast mode.
  • The input[type=number] element provides mobile assistive technology users a means to change the temperature when an external keyboard is not being used.
  • The button elements provides mobile assistive technology users a means to change the Fan and Heat/Cool values when an external keyboard is not being used.
  • The button elements receive a border style when hovered for users to identify the elements as interactive. Border is not rendered by default since the content of the buttons is primarily as visual labels for values of the slider positions. The buttons are only a secondary way to set the value of the slider.

Keyboard Support

Slider

Key Function
Right Arrow Increases slider value one step.
Up Arrow Increases slider value one step.
Left Arrow Decreases slider value one step.
Down Arrow Decreases slider value one step.
Page Up Increases temperature slider value multiple steps. In this slider, jumps ten steps.
Page Down Decreases temperature slider value multiple steps. In this slider, jumps ten steps.
Home Sets slider to its minimum value.
End Sets slider to its maximum value.

Buttons

Key Function
Space, Enter
  • The buttons support mobile devices when a keyboard is not available.
  • Pressing the space key or clicking on the button changes the value of the slider referenced by aria-controls to the value in the data-value attribute of the button element.

Role, Property, State, and Tabindex Attributes

Slider

Role Attribute Element Usage
slider g
  • Identifies the element as a slider.
  • Set on the div that represents as the movable thumb because it is the operable element that represents the slider value.
tabindex=0 g Includes the slider thumb in the page tab sequence.
aria-orientation g
  • Indicates the orientation of the slider element.
  • Set to vertical for the temperature slider.
  • Set to horizontal for the fan and heat/cool controls.
aria-valuemax=NUMBER g Specifies a numeric value that is the maximum value the slider can have.
aria-valuemin=NUMBER g Specifies a numeric value that is the minimum value the slider can have.
aria-valuenow=NUMBER g A numeric value that is the current value of the slider.
aria-valuetext=STRING g
  • A string value that provides a user-friendly name for the current value of the slider.
  • For the temperature slider, is the value of the slider appended with the text degrees.
  • For fan and heat/cool, is the text representation of the current state, e.g., Off or Heat.
aria-labelledby=IDREF g Refers to the element containing the name of the slider.
aria-hidden=true rect Removes the SVG rect element from the accessibility tree, since some screen readers will describe it as an image separate from the slider.

Input

Role Attribute Element Usage
tabindex=-1 input
  • The input[type=number] supports mobile access when an external keyboard is not available to operate the slider control directly. Mobile devices provide an on-screen keyboard when input boxes receive focus for entering a temperature value.
  • Removes the input element from the tab order of the page.

Buttons

Role Attribute Element Usage
tabindex=-1 button
  • The button supports mobile access when an external keyboard is not available to operate the slider directly. Mobile devices provide an on-screen keyboard when input boxes receive focus for entering a temperature value.
  • Removes the button element from the tab order of the page.
aria-label=label button Defines the accessible name for each button describing how the associated slider will change (e.g. "Heating off", "Fan off" and "Fan auto").
aria-controls=IDREF button
  • Identifies the content on the page that the button controls.
  • Refers to the SVG g element that contains identifies the slider.

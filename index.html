ARIA Authoring Practices Guide Home

Learn to use the accessibility semantics defined by ARIA to create accessible rich internet applications. This guide describes how to apply accessibility semantics to common design patterns and widgets. It provides design patterns and functional examples complemented by in-depth guidance for fundamental practices.

Get Involved

The APG Task Force relies on broad community representation and participation to continuously improve the usefulness and quality of the APG. There are a variety of ways you can get involved and help promote development of accessible experiences.

  • Join the Task Force

    To join the APG Task Force, individuals need to first join the W3C ARIA Working Group. Participants are expected to actively contribute to the work of the task force.

    Learn more about the work of the task force and how to join

  • Contribute via GitHub

    Many valueable contributions are made by people who find or raise issues of interest in our GitHub repository and then submit proposed changes via a GitHub pull request. If you choose this path, it is tremendously helpful if you first study the APG's contribution guidelines.

    View ReadMe in the GitHub repository

  • Mailing Lists

    The APG Task Force uses the public-aria-practices@w3.org mailing list (view the mailing list archive) for email discussion.

    Discussions of the task force prior to September 2017 are archived on the public-aria mailing list.
